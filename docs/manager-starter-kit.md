# FLH Manager Starter Kit

Version: 0.1.0

## Purpose

This guide helps a department manager start using FLH agent governance without designing a new system from scratch.

It is not a new policy layer. It is a practical adoption path that sits on top of:

- `skill-governance.md`
- `kb-governance.md`
- `knowledge-card-governance.md`

## What To Use It For

- starting a department-level pilot
- deciding which skill or KB to standardize first
- giving team members a safe baseline to copy and adapt
- avoiding one-off governance decisions in every team

## Minimum Starting Stack

1. Pick one manager-owned use case.
2. Choose one skill package or one KB to pilot first.
3. Copy the governance baseline from `docs/`.
4. Assign one owner.
5. Set one review cadence.
6. Record what changed in a short log.

## Recommended First Pilot Questions

- What problem are we trying to make easier?
- Who owns the asset?
- What is the smallest version that is still useful?
- What must be reviewed before sharing?
- What should stay private?

## Suggested Department Rollout

### Phase 1: Copy The Baseline

- use the published governance docs as the default template
- do not invent a custom format unless there is a real gap
- keep the first version small

### Phase 2: Adapt For The Department

- tighten naming conventions if needed
- choose the review owner
- define what counts as sensitive
- decide how the team will share updates

### Phase 3: Standardize What Works

- promote the working version into a department norm
- keep the original published governance as the baseline
- avoid diverging formats unless there is a clear reason

## What Not To Do

- do not create a separate governance system for every team
- do not start from empty folders without an owner
- do not mix drafts with published policy
- do not overdesign before the first pilot works

## Quick Start

1. Read `skill-governance.md` or `kb-governance.md`.
2. Pick the one that matches your first pilot.
3. Copy the checklist into your team workflow.
4. Assign an owner and a reviewer.
5. Start with one concrete asset, not a full library.
