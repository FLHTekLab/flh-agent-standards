# docs Guide

`docs/` stores repository-level methods, rules, workflows, and stable guidance.

Use `docs/` when content:

- can be reused across multiple tasks
- serves as a stable reference
- explains a repository-level governance rule

Do not put these here first:

- task-specific notes
- early drafts
- execution artifacts

## Current Structure

### `skill-governance.md`

Rules for FLH agent skill packages, versioning, maintenance, and release flow.

### `kb-governance.md`

Rules for FLH agent knowledge bases, layer contracts, sharing levels, and lifecycle.

### `knowledge-card-governance.md`

Rules for short reusable knowledge cards and promotion into canonical guidance.

### `manager-starter-kit.md`

A practical adoption path for department managers who want to start from the published governance baseline instead of designing a new process.

## Reading Order

1. Read `docs/README.md`.
2. Enter the relevant governance document.
3. Return to root `README.md` or `ARCHITECTURE.md` if you need repository boundaries.
